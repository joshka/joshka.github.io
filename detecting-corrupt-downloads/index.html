Detecting and Preventing Corrupt Software Downloads

2010-04-30
:: tags:  #Software

In the past I and several of my colleagues have run into problems when attempting to install software downloaded from the MSDN Subscriber Downloads or elsewhere. One possible cause for this is file corruption either due to an incomplete download, or caused later when copying the file to another device (e.g. over a local network or via flaky USB drives). This problem is easy to detect and prevent.

  1. Install the HashCheck Shell Extension by Kai Liu. There are other options for hash checking programs, but this one works well enough.
  2. On the download page for the file, copy the SHA1 hash. For MSDN downloads this can be found by expanding the Details View link.
  3. In Windows Explorer open the Properties dialog for the file that you downloaded. Click on the new Checksums tab. HashCheck will calculate the various checksums of the file.
  4. Paste the hash into the text box on the Checksums tab and ensure that the SHA1 line is highlighted to indicate that the values match.
  5. Click Save to create a new .sha1 file beside the downloaded file that contains this calculated and checked hash. You can double click this file to check whether the file has changed since you downloaded it. Do this whenever you transfer the file on to another device (e.g. network, DVD, USB) to prevent file corruption causing installation failures.
